# 代码审查检查清单

## 通用检查

### 代码质量
- [ ] 代码遵循 Kotlin 编码规范
- [ ] 没有硬编码的字符串（使用 strings.xml）
- [ ] 没有魔法数字（使用常量）
- [ ] 变量和函数命名清晰有意义
- [ ] 代码格式化正确（使用 ktlint）
- [ ] 没有未使用的导入
- [ ] 没有注释掉的代码

### 架构
- [ ] 遵循 Clean Architecture 分层
- [ ] 正确使用 MVVM 模式
- [ ] UseCase 只包含业务逻辑
- [ ] Repository 正确抽象数据源
- [ ] ViewModel 不包含 Android 依赖

### 错误处理
- [ ] 所有异常都被正确捕获
- [ ] 错误信息对用户友好
- [ ] 网络错误有重试机制
- [ ] 有适当的日志记录
- [ ] 边界情况都被处理

### 性能
- [ ] 没有内存泄漏
- [ ] 避免在主线程执行耗时操作
- [ ] 图片加载优化
- [ ] 列表使用 LazyColumn
- [ ] 避免不必要的重组

### 安全
- [ ] 没有硬编码的密钥或密码
- [ ] 敏感数据加密存储
- [ ] 使用 HTTPS
- [ ] 输入验证完整
- [ ] SQL 注入防护（如适用）

### 测试
- [ ] 有单元测试
- [ ] 测试覆盖率 > 70%
- [ ] 关键功能有 UI 测试
- [ ] 所有测试都通过
- [ ] 测试命名清晰

## Android 特定检查

### UI/UX
- [ ] 支持深色模式
- [ ] 支持不同屏幕尺寸
- [ ] 支持横竖屏切换
- [ ] 加载状态有提示
- [ ] 错误状态有提示
- [ ] 空状态有提示

### Compose
- [ ] 使用 remember 缓存状态
- [ ] 使用 key 优化列表
- [ ] 避免在 Composable 中执行副作用
- [ ] 使用 LaunchedEffect 处理副作用
- [ ] 参数使用 @Stable 或 @Immutable

### 资源
- [ ] 字符串都在 strings.xml
- [ ] 颜色都在 colors.xml
- [ ] 尺寸都在 dimens.xml
- [ ] 图片资源优化（WebP）
- [ ] 支持多语言

### 权限
- [ ] 只请求必要的权限
- [ ] 运行时权限处理正确
- [ ] 权限拒绝有提示
- [ ] 权限说明清晰

### 网络
- [ ] 使用 HTTPS
- [ ] 有超时配置
- [ ] 有重试机制
- [ ] 有缓存策略
- [ ] 错误处理完整

### 数据存储
- [ ] 敏感数据加密
- [ ] 使用正确的存储方式
- [ ] 数据迁移处理
- [ ] 备份和恢复
